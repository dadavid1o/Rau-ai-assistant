CLI-ассистент для работы с учебными планами и УМКД магистерских программ RAU.  
Проект объединяет данные по дисциплинам в единой SQLite-базе и позволяет быстро искать информацию по курсам: название, код, семестр, кредиты, описание, результаты обучения и компетенции.

## Problem

Учебные планы и УМКД содержат большое количество важной информации:
- названия дисциплин,
- коды курсов,
- семестры,
- кредиты,
- краткие описания,
- результаты обучения,
- компетенции.

На практике эти данные хранятся в PDF-файлах и таблицах, поэтому поиск нужной информации вручную неудобен и занимает много времени.  
Пользователю приходится открывать несколько документов, пролистывать страницы и самостоятельно сопоставлять учебный план, код дисциплины и содержимое УМКД.

## Goal

Цель проекта — создать CLI-ассистента, который позволяет быстро находить информацию по дисциплинам магистратуры на основе структурированной базы данных.

Система должна:
- хранить сведения о дисциплинах в едином формате,
- поддерживать поиск по коду, названию и ключевым словам,
- помогать быстро получать информацию о курсе без ручного просмотра PDF-документов,
- служить основой для дальнейшего перехода к полноценственной RAG-архитектуре.

## Solution Approach

Для решения задачи используется следующий подход:

1. **Сбор и структурирование данных**
   - данные из учебных планов и УМКД переводятся в структурированный вид;
   - каждая дисциплина описывается набором полей:
     - plan
     - index_code
     - name
     - semester
     - credits
     - course_type
     - description
     - learning_outcomes
     - competencies
     - source

2. **Хранение данных**
   - в качестве основной базы используется **SQLite**;
   - база хранит как “скелет” дисциплин из учебного плана, так и содержательные данные из УМКД.

3. **Импорт данных**
   - исходные данные из учебных планов подготавливаются в CSV;
   - затем база дополняется содержимым из УМКД:
     - краткое описание курса,
     - результаты обучения,
     - компетенции,
     - источник данных.

4. **Поиск**
   - реализован поиск по:
     - коду дисциплины,
     - названию курса,
     - алиасам и ключевым словам;
   - пользователь может быстро находить нужную дисциплину в рамках конкретного учебного плана.

5. **Формирование ответа**
   - после нахождения подходящей записи ассистент выводит найденный контекст;
   - при наличии OpenAI API система может формировать более естественный текстовый ответ на основе найденных данных.

6. **Воспроизводимость**
   - обновления по УМКД сохраняются в SQL-файлах;
   - это позволяет пересоздавать и восстанавливать содержимое базы без ручного повторного ввода.

## Current Result

На текущем этапе проект уже поддерживает две магистерские программы:
- **MatMod**
- **MLDS**

Сейчас реализовано:
- хранение дисциплин в SQLite;
- поиск по базе через CLI;
- поэтапное заполнение полей `description`, `learning_outcomes` и `competencies` из УМКД;
- сохранение SQL-обновлений для повторного применения к базе.

Для обеих программ подготовлены отдельные SQL-файлы с чистыми обновлениями:
- `data/sql/matmod_umkd_updates_clean.sql`
- `data/sql/mlds_umkd_updates_clean.sql`

Это позволяет восстановить содержимое базы после пересоздания без ручного повторного ввода.

## Project Structure

- `src/` — основной код приложения  
  - `app.py` — CLI-интерфейс  
  - `db.py` — работа с SQLite  
  - `import_csv.py` — импорт данных из CSV  
  - `llm.py` — работа с OpenAI API  

- `data/` — данные проекта  
  - `matmod_skeleton.csv` — базовый список дисциплин MatMod  
  - `mlds_skeleton.csv` — базовый список дисциплин MLDS  
  - `sql/` — SQL-обновления для заполнения базы из УМКД  

- `ucheb_planer/` — PDF-файлы учебных планов  
- `courses.db` — основная SQLite-база данных  
- `README.md` — описание проекта  
- `tools_extract_mlds.py` — вспомогательный скрипт для работы с данными MLDS
